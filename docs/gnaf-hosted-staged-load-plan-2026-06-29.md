# Hosted G-NAF staged load plan

Date: 2026-06-29

## Decision

Use staged hosted G-NAF loading. Do not run the full national hosted load until shard evidence is held and the national cost is explicitly approved.

## Stage 0: current state

- Local national SQLite exists and is nationally complete for local mechanics.
- Hosted Neon target exists and currently has preview rows.
- Full national hosted load is not approved.

## Stage 1: validation load

Purpose: prove the raw loader still works on the current ZIP and target.

Command:

```bash
npm run load:gnaf-raw-postgres -- --input data/gnaf/raw/g-naf_may26_allstates_gda2020_psv_1023.zip --limit-per-state 1000
```

Pass criteria:

- Command completes.
- Hosted readiness can query the table.
- No missing expected indexes after index creation.

## Stage 2: NSW state-shard load

Purpose: prove large-shard storage growth, write time and query behaviour before national load.

Why NSW:

- NSW is the largest local state shard at 5,206,855 rows.
- If NSW behaves well, smaller states are lower risk.
- If NSW behaves badly, national load should not proceed.

Command:

```bash
npm run load:gnaf-raw-postgres -- --input data/gnaf/raw/g-naf_may26_allstates_gda2020_psv_1023.zip --states NSW --reset --skip-indexes --allow-state-shard --storage-review docs/gnaf-hosted-storage-review-2026-06-29.json --progress-json tmp/gnaf-raw-postgres-load-NSW-shard.json --run-id NSW-shard
npm run load:gnaf-raw-postgres -- --setup-only --create-indexes
npm run check:gnaf-hosted:readiness
```

Pass criteria:

- Load completes without manual database repair.
- Progress JSON records completion and row count.
- Index creation completes.
- Hosted readiness passes for the shard.
- Storage growth is recorded before and after indexes.
- Query benchmark passes for NSW exact, unit, suburb, rural, remote and POI-like cases.

## Stage 3: shard benchmark

Purpose: decide whether the hosted index is good enough for users.

Required benchmark slices:

- exact addresses
- unit and slash addresses
- townhouse/building-style addresses
- rural and remote towns
- suburb/locality queries
- POI-like ambiguous queries
- numeric-address-like queries

Metrics to capture:

- top result correctness
- any result correctness
- p50 latency
- p90 latency
- p95 latency
- slowest query
- provider used
- query category

## Stage 4: rollback check

Purpose: prove we can undo a bad hosted load.

Minimum rollback path:

- preserve progress JSON
- preserve previous deployment/config state
- keep `FUEL_PATH_GNAF_STORAGE_REVIEW_CONFIRMED` unset
- reset/reload preview rows if shard result is rejected

Brutal critique:

- We do not yet have a first-class automated rollback command.
- Until rollback is scripted, national load should remain blocked.

## Stage 5: national load

Only run after all of these are true:

- NSW shard load passes.
- NSW shard benchmark passes.
- Storage expansion is measured and accepted.
- Monthly national cost is accepted.
- Rollback path is documented or scripted.
- Hosted national benchmark command is ready.
- `FUEL_PATH_GNAF_STORAGE_REVIEW_CONFIRMED=1` is set explicitly.

National command should be generated by:

```bash
npm run plan:gnaf-hosted-load
```

## Current blockers

- Full national hosted storage cost is not approved.
- NSW state-shard load failed on the current hosted Neon target after 1,655,000 rows because the project size limit is 512 MB.
- State-shard benchmark is not yet run because the shard load did not complete.
- Automated rollback is not yet implemented.
- Hosted 900-case national benchmark is not yet held.

## 2026-06-29 NSW shard attempt

The NSW shard load was attempted with the approved staged command and failed before completion.

Evidence:

- Result file: `docs/gnaf-hosted-nsw-shard-load-result-2026-06-29.md`
- Progress file: `tmp/gnaf-raw-postgres-load-2026-06-29T02-52-32-960Z-NSW-shard.json`
- Rows loaded before failure: `1,655,000`
- Error: `could not extend file because project size limit (512 MB) has been exceeded`

Rollback:

- Restored preview-sized hosted dataset with `80,000` rows.
- Recreated expected hosted search indexes.
- Hosted readiness remains blocked only by row count after rollback.
- The first rollback attempt without explicit `--states` stuck on the `OT` source bucket, so future rollback/load commands should pass explicit supported states.

Decision:

- Do not retry the shard load on the current storage tier.
- Do not run national load.
- Do not run hosted 900-case benchmark.
- Do not immediately upgrade storage just to repeat the raw-table load.
- Trial the compact serving index first: `docs/gnaf-compact-serving-index-plan-2026-06-29.md`.
- Fix storage tier or architecture only after compact NSW storage evidence is held.
