# Local Vercel deploy cleanup path

Date: 2026-06-29

## Decision

Use a repo-local npm cache for Vercel CLI deploys from this machine.

The global npm cache has previously produced permission errors. The local cache path is already ignored by Git and is now ignored by Vercel upload packaging, including nested cache folders.

## Preferred production deploy command

```bash
sh scripts/deploy-vercel-production.sh
```

The wrapper runs:

```bash
npm_config_cache=.npm-cache npx vercel --prod --yes
```

Additional Vercel CLI flags can be passed through:

```bash
sh scripts/deploy-vercel-production.sh --scope <team-or-user>
```

## Cleanup rules

- Keep `.npm-cache/` local.
- Keep `api/.npm-cache/` local.
- Keep `mobile-app/.npm-cache/` local.
- Keep `.vercel/` local.
- Do not commit `public/`; it is generated by the Vercel build.
- Do not upload `tmp/`, local G-NAF data, SQLite files, raw ZIPs or local provider samples.

## Vercel upload ignore coverage

`.vercelignore` now excludes:

- `.npm-cache/`
- `**/.npm-cache/`
- `.vercel/`
- `tmp/`
- `var/`
- `/data/`
- SQLite files
- ZIP files
- dependency folders

## Brutal critique

- This does not fix the underlying global npm cache permissions. It routes around them safely.
- This does not fix GitHub push authentication. Browser login helps with GitHub web access, but local `git push` still needs local Git credentials, SSH auth or another approved push route.
- This does not guarantee production readiness. It only makes local deploy execution repeatable.
- If Vercel CLI auth expires, the wrapper will still fail and the user must re-authenticate Vercel locally.

## Pass criteria

- Deploy wrapper exists.
- Deploy wrapper uses repo-local npm cache.
- Vercel ignore excludes nested npm caches and `.vercel/`.
- Documentation records the repeatable production deploy path and remaining auth limits.
